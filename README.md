# 📊 Test Video Stats Bot

Telegram-бот для анализа статистики просмотров видео на естественном русском языке.

Бот загружает данные из файла videos.txt в базу данных PostgreSQL, принимает от пользователя вопросы о статистике просмотров и отвечает на них, автоматически генерируя и выполняя SQL-запросы через LLM.

---

## 🚀 Возможности
- 📥 Загрузка данных о видео из файла videos.txt
- 💬 Обработка вопросов на естественном русском языке
- 🧠 Генерация SQL-запросов с помощью LLM
- 🔍 Выбрать подходящую модель можно самостоятельно
- 🔐 Повторная валидация SQL-запроса через LLM для безопасности
- 🗄 Выполнение SQL-запроса к базе данных
- 📊 Формирование понятного ответа пользователю

Пример вопросов, которые можно задать боту:
- "Какое видео набрало больше всего просмотров?"
- "Сколько всего просмотров за март?"
- "Среднее количество просмотров по всем видео?"
- "Топ-5 самых популярных видео"

## 🐳 Запуск через Docker

### 1. Клонировать репозиторий

```shell
git clone https://github.com/Cherimolah/test_video_stats_bot.git
cd test_video_stats_bot
```

### 2. Создать .env файл (см. [Переменные окружения](#переменные-окружения))
```shell
cp .env.example.env
nano .env
```

### 3. Запустить Docker образ
```shell
docker-compose up -d
```

## Переменные окружения
| Имя переменной   | Описание                                                                          |
|------------------|-----------------------------------------------------------------------------------|
| TELEGRAM_TOKEN   | Токен телеграм бота                                                               |
| OPENROUTER_TOKEN | Токен с сайта [OpenRouter](https://openrouter.ai/)                                |
| DB_USER          | Имя пользователя PostgreSQL                                                       |
| DB_PASSWORD      | Пароль от PostgreSQL                                                              |
| DB_NAME          | Имя базы данных                                                                   |
| DB_HOST          | Хост базы данных. В `docker-compose.yml` указан как `db`. Рекомендуется не менять |
| DB_PORT          | Порт базы данных                                                                  |
| MODEL_NAME       | Имя модели. Можно оставить по умолчанию `arcee-ai/trinity-large-preview:free`     |
